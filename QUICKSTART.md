# AI Ready Suite — 快速入门

> 从安装到产出第一份报告，约 10 分钟。

---

## 1. 安装

```bash
# 进入插件目录
cd ai-ready-suite

# 安装 MCP 引擎依赖
cd mcp-servers/ai-ready-engine && npm install && cd ../..

# 验证安装
node test/run-assessment.js
node test/integration-test.js
```

如果测试全部通过（✅ 54/54），安装完成。

---

## 2. 首次体验：评估模式（5 分钟）

### 准备数据

复制示例数据到你的工作空间：

```bash
cp examples/sample-components.json /your/workspace/sample-components.json
```

### 启动评估

对 Codex 说：

> **"帮我评估一下这个组件库的 AI 就绪度"**

或者，更直接一点：

> **"评估 sample-components.json 的 AI 就绪度"**

### 对话流程

引擎会引导你完成这些步骤：

| 步骤 | 你做什么 | 引擎做什么 |
|------|---------|-----------|
| 场景选择 | 选评估目的（回答老板/找差距/立项/汇报） | 记录场景，推荐角色 |
| 路径选择 | 选快速诊断(5min) 或 完整评估(20-30min) | 提示时间预期 |
| D0 预处理 | 无需操作 | 自动分析组件数据：分类/Variant 归一化/Token 绑定 |
| D1-D7 确认 | 看引擎建议分，可选调整 | 展示各维度自动评分 |
| D8-D10 问答 | 回答 3 个问题 | 引导回答并计算分数 |
| 输出报告 | 获得评估结果 | 生成卡片报告 + JSON |

### 示例输出

```

║  AI Ready 评估报告                        ║
╠══════════════════════════════════════════╣

║  等级：🟢 L2 AI 就绪                      ║
║  ...                                      ║
╚══════════════════════════════════════════╝
```

## 3. 审计模式（3 分钟）

准备好一段代码或设计输出文本，然后对 Codex 说：

> **"审计一下这段 AI 生成的代码"**

或者用示例数据：

```
把 examples/sample-code-to-audit.js 的内容发给 Codex，说"看看这段代码质量"
```

引擎执行 C/B/A 三层检查（共 25 项），返回通过率 + 问题清单。

---

## 4. 侦察模式（2 分钟）

如果你有组件库 JSON 数据，对 Codex 说：

> **"侦察一下这个组件库"**

引擎会执行：发现库 → 拉取数据 → 验证质量（5 项检查：数量一致性、文件名规范性、JSON 可解析性、组件名唯一性、Props 定义完整性）。

---

## 5. 常见问题

| 问题 | 解决 |
|------|------|
| "评估没有返回" | 检查数据是否包含 `components` 数组 |
| "MCP 工具调用失败" | 执行 `npm run test` 确认引擎正常运行 |
| "所有分数都是 3.0" | 在对话中逐一回答 D1-D10 的问题 |
| "信心指数低" | 补充组件的 description 和 Props 定义 |

## 8. 卸载

### 从 Codex 中移除

1. 打开 Codex 设置 → 插件管理
2. 找到 **AI Ready Suite**，点击移除
3. 插件市场中将不再显示该插件

### 删除本地文件

```bash
# 确认插件目录路径后执行
rm -rf /path/to/ai-ready-suite
```

> **注意**：删除前请备份你的评估报告和自定义组件清单。插件本身不存储用户数据，但 `examples/` 目录下的自定义修改需要手动备份。

 ### 激活码处理
 
 如果已激活 Pro，`pro-key.json` 会随插件目录一并删除。如需保留激活状态，请备份该文件到安全位置。
 
 ### 确认卸载完成

```bash
# 检查插件目录是否已清空
ls /path/to/ai-ready-suite  # 应返回 "No such file or directory"
```
